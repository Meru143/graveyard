# Security Policy

## Supported Versions

Security fixes are applied to the latest released `0.1.x` line. Older versions may receive documentation-only guidance but are not guaranteed to receive patched artifacts.

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |
| < 0.1.0 | No |

## Reporting a Vulnerability

Do not open public GitHub issues for security-sensitive reports.

Use GitHub Private Vulnerability Reporting for this repository when it is available. If private reporting is not enabled in your environment, contact the maintainers through the repository owner channel listed on GitHub and include enough detail to reproduce the issue, the affected version, any proof-of-concept material, and whether the problem is already public.

You can expect an initial acknowledgement within five business days. After triage, maintainers will communicate impact, remediation scope, and a coordinated disclosure timeline.

## Scope

This policy covers the released `graveyard` binary, packaging metadata, GitHub Actions workflows, and repository-owned installation artifacts. Vulnerabilities in third-party dependencies may be addressed through dependency updates, deny-listing, or advisory documentation depending on exploitability and release impact.
